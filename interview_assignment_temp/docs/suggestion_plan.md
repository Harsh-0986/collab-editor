After reading the assignment a few times, I think there's one thing that would put your submission above almost everyone else's:

Build a "Developer Diagnostics" panel.

Not just a sync indicator—a panel that visualizes the synchronization engine.

Imagine a collapsible drawer showing:

Connection status (Online/Offline)
Queue length
Pending operations
Current Lamport clock
Local version vs. server version
Last sync duration
Retry count
Active client ID
Recent synchronization log
"Force Sync" button

When the reviewer opens DevTools and sees a transparent, inspectable synchronization engine, it immediately communicates that you understand distributed systems rather than having copied a rich-text editor.

That single feature aligns almost perfectly with the assignment's emphasis on offline synchronization, conflict resolution, and engineering thoughtfulness. It also makes your architecture much easier to demonstrate during an interview.
